I Replaced Chrome DevTools MCP with Safari and My MacBook Stopped Overheating

Three weeks ago I made a change that cut my MacBook Pro's average CPU temperature by roughly 15 degrees Celsius during work hours. I didn't buy a cooling pad. I stopped using Chrome.

Not for browsing — I hadn't used Chrome for browsing in years. I stopped using it as the invisible backbone of my AI coding workflow.

If you use Claude Code, Cursor, or any MCP-enabled AI tool with browser automation, there's a good chance Chrome is running in the background right now, eating 8–15% of your CPU.

The Setup That Was Cooking My Laptop

I run an automation consulting business. My daily workflow involves Claude Code connected to several MCP servers. The browser MCP server was Chrome DevTools MCP, which works by launching Chrome with a remote debugging port.

Chrome with a debug port open idles at 8–15% CPU on an M2 MacBook Pro. During active automation it spikes to 25–40%. Chrome alone was consuming 200–400MB of RAM before a single page loaded.

Safari was already open. All my logins were there. Gmail, GitHub, AWS console, client dashboards. I was running two browsers while the browser I actually needed was right there, already authenticated.

Why Safari?

Apple doesn't give you Chrome's hooks. What Safari does have is AppleScript — you can run arbitrary JavaScript in any Safari tab from an external process. It gives you access to the actual DOM of the actual browser where you're logged in.

That was the starting point. It became Safari MCP — 80 tools for native Safari browser automation via AppleScript + a Swift daemon.

What Changed

The first thing I noticed was silence. The fans stopped. Safari MCP uses about 30MB of memory. Idle CPU sits at 0.1%. During active automation it bumps to 2–5%.

I stopped re-authenticating. The agent uses my real Safari with all my existing sessions. GitHub logged in. AWS logged in. Client portals logged in.

Safari stopped stealing my focus. Every command targets a specific tab by index. Tabs open in the background. Clicks target specific window IDs without moving the cursor.

Getting Started

npm install safari-mcp
